# Git4Gits

**Learning platform** for **subject matter experts** mastering **structured, verifiable AI-assisted coding**—not generic LMS shelfware. This repository holds **product vision, pedagogy, intake flows, rubrics, jurisdiction scaffolding, and intelligence design**; app code will live under [`apps/`](apps/README.md) or a sibling repo.

The **thrill** is superpower-first learning; the **moat** is verification habits, calibration, and visual product craft. Anchors: [`plan/platform-product-vision.md`](plan/platform-product-vision.md), [`plan/product-shape.md`](plan/product-shape.md), [`plan/messaging-unaware-learners.md`](plan/messaging-unaware-learners.md).

## Start here

1. Read **[`plan/platform-product-vision.md`](plan/platform-product-vision.md)** — what we’re building as a platform.
2. Open [`plan/00-start-here.md`](plan/00-start-here.md) for locked-in pilot defaults and facilitator actions (`product-shape.md` reflects platform thesis).
3. Scan [`plan/domains.md`](plan/domains.md) and [`plan/research-ai-disruption-notes.md`](plan/research-ai-disruption-notes.md) when framing cohort industry focus.
4. If learners sit under **meaningful jurisdiction / sector overlay**, run desk research via [`plan/compliance-research-agent.md`](plan/compliance-research-agent.md); pin results in `plan/legal-packs/` — **not legal advice**; escalate to counsel when red flags appear.
5. For each enrolled SME before Week 1: **`templates/learner-intake-form.md`** (typed async) → [`plan/learner-intake-interview.md`](plan/learner-intake-interview.md) optional live pass → profile in [`plan/learners/README.md`](plan/learners/README.md). Elite roles: **[`plan/conversation-intelligence-prep.md`](plan/conversation-intelligence-prep.md)** + `plan/prep-briefs/`.
6. Run the **Week 1 checklist** in the same folder.
7. Use [`templates/problem-brief-template.md`](templates/problem-brief-template.md) before any codegen in teaching or pilot sessions.
