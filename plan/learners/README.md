# `plan/learners/` — individual learner profiles

## Purpose

One Markdown file **per enrolled SME**, produced by [`learner-intake-interview.md`](../learner-intake-interview.md) together with [`../templates/learner-profile-template.md`](../templates/learner-profile-template.md).

Tracks **domain authority**, **industry**, **country / sub-jurisdiction**, and **passion** so capstones and facilitation match the person—not an abstract persona.

## How to create a file

1. Copy [`../templates/learner-profile-template.md`](../templates/learner-profile-template.md).
2. Save as `plan/learners/<anonymous-slug>.md` — e.g. `fpna-chicago-001.md`, not legal names, if repo is public or shared.
3. If your employer forbids learner detail in Git, keep profiles on an **internal wiki** and leave only stub pointers here.

## PII & secrecy

Assume profiles contain **sensitive role and geography** signals. Prefer **minimal identifiers** and sponsor-approved storage. Never paste customer secrets or credentials into profiles.
